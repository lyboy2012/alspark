  • 函数组件

  • 其他分类形式

    • 无状态 （函数组件、类组件都能实现无状态组件）
    • 有状态 （类状态组件、useState hooks 可以实现有状态组件）

  • 组件渲染

    更新 UI 唯一的方式是创建一个全新的元素，并将其传入 ReactDOM.render()；
    组件更新UI 形式可以通过setState 触发render 来实现。

  • props

    Props are Read-Only 属性是只读的
    组件无论是使用函数声明还是通过 class 声明，都决不能修改自身的 props。

    • props放什么数据？
      • 组件内部不变的数据；
      • 需要外部传入不可改变的数据（仅UI显示固定值或是作为内部判断逻辑参数）。

state & 生命周期

  • state
    组件静态不变数据放到props上，组件内动态变化数据放到state里。
    state 修改后会触发render方法，实现重绘。
  • 注意问题：
    • state只能通过setState来修改；
    • setState是异步的不会立即修改state值；
    • 多次调用setState为了提高性能会合并。
  • 生命周期
    组件在初始化、渲染、销毁等过程中提供的钩子函数，方便做些而外逻辑处理等。

事件处理

react 事件通过统一封 SyntheticEvent 包装器来实现；

  • 实现原理(事件代理)

    真实事件绑定在document上，基于浏览器的事件机制（冒泡），交给SyntheticEvent 包装器统一处理。

  • 好处：

    • 减少内存的消耗，提高性能；
    • 统一规范解决兼容问题；

单向数据流

react是“自上而下”的“单向”的数据流。

  • 数据传递
    任何的 state 总是所属于特定的组件，且从该 state 派生的任何数据或 UI 只能影响树中“低于”它们的组件。
    说人话：数据只能通过props从父组件传递给子组件，反过来是不行的。

    单向数据流引起reRender

  • 父子组件通信方式

    通过props把修改父组件state回调函数传个子组件，来实现修改父组件state，以达到子和父的通信。
